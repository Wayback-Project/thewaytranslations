# YHWH / Elohim / God Harmonization Matrix (Working)

Created: 2026-04-08 UTC
Status: Active policy matrix for transparent naming decisions

## Goal
Keep source-name fidelity while avoiding inconsistent drift across OT/NT tracks.

## Core rule
- Preserve source names where recoverable and contextually clear.
- Do not flatten YHWH and Elohim into one label by default.

## Matrix

| Corpus zone | Source priority | Preferred rendering | Notes |
|---|---|---|---|
| Torah / Prophets / Writings (Hebrew-first OT) | Hebrew | `YHWH`, `Elohim`, `El`, etc. preserved | Keep distinctions explicit; no generic LORD substitution. |
| Aramaic OT passages | Aramaic | Preserve Aramaic divine title where text supports | Keep transliteration consistency notes per passage. |
| Gospels / sayings-heavy NT (Way One Aramaic-resonance track) | Aramaic-primacy posture + Greek check | `Elohim` generally preferred in divine-name/theological force lines | If Greek-critical constraints require, document exceptions. |
| Acts (dual witness) | Mixed | `Elohim` default in Semitic-context lines; `God` allowed in Greek-formulaic speech where necessary | Log rationale for shifts. |
| Pauline-and-later NT (Greek-grounded acknowledged track) | Greek-critical | `God` default in formulaic epistolary lines | Keep issue-based review for larger harmonization and Father/Cosmic Parent policy boundaries. |

## Style guardrails
1. Never replace explicit `YHWH` with `LORD` in this project.
2. When `Elohim` is chosen in NT, ensure it is intentional and documented (not accidental carryover).
3. Where `God` is retained in Greek-grounded late NT, log that as track-based, not doctrinal flattening.
